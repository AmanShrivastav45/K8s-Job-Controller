01 The Problem MCP Solves

Before MCP, connecting an AI model to external tools was fragmented and one-off. Every combination of AI frontend and tool required its own custom integration — built from scratch, reusable nowhere.

This is the N×M problem: N tools × M AI frontends = N×M custom integrations.

🍕 Analogy — The Delivery Bike Problem

Imagine every restaurant in the city had to buy and maintain their own fleet of delivery bikes — one per food app (Swiggy, Zomato, Dunzo). That's insane. Instead, Zomato's network works with all restaurants through one standard. MCP is that delivery network for AI tools.

MCP collapses this to N + M. Build a server once, and any MCP-compatible AI can use it — without rewriting the integration for every pairing.

02 Core Concepts

What is MCP?

MCP (Model Context Protocol) is an open standard — not a library, not a SaaS product — for how AI models communicate with external tools and data sources. Think of it like REST or GraphQL, but designed specifically for AI agents to consume. It uses JSON-RPC 2.0 as its wire format — the same standard that powers IDE language intelligence (LSP).

🔌 Analogy — USB-C

Before USB-C, every device had its own charger. MCP is USB-C for AI tools. It doesn't matter if the laptop is a Dell or a MacBook — one cable works everywhere. Similarly, it doesn't matter if your AI is Claude or GPT — one MCP server works with both.

The Three Actors

ComponentWho It IsWhat It Does
MCP HostClaude Desktop, Cursor, your appThe AI application the user talks to. Contains the LLM and orchestrates everything.
MCP ClientLives inside the HostManages a 1:1 session with one MCP Server. Handles connections, errors, and timeouts.
MCP ServerYour tool / external serviceExposes capabilities to the AI. Translates external responses into a format the model understands.

What an MCP Server Exposes

  • Tools — Executable actions with side effects (call an API, run a DB query, send an email). The model decides when to invoke these.
  • Resources — Read-only data retrieval from files or databases. No side effects. Context the model can pull in.
  • Prompts — Reusable, parameterised templates for structured LLM workflows.
🏥 Analogy — A Hospital Front Desk

The MCP Server is like a hospital receptionist. Tools are things they can do (book an appointment, send a referral). Resources are things they can show you (test results, medical history). Prompts are the standard scripts they follow for common situations ("first-time patient intake").

03 How It Works — Step by Step

Concrete example: you ask an AI — "Find the latest sales report and email it to my manager."

#StepWho Acts
1User sends the request to the AI application.User → MCP Host
2LLM processes the request and determines it needs external tools.MCP Host (LLM)
3MCP Client queries connected servers: "What tools do you expose?"MCP Client
4LLM reads tool descriptions and decides which to call and in what order — no developer hard-codes this logic.MCP Host (LLM)
5Tool calls are sent via JSON-RPC 2.0 to the MCP Server, which executes and returns results.MCP Server
6LLM synthesises results and responds to the user naturally.MCP Host (LLM)
💡 Key Insight

The LLM is the decision-maker in the middle — it reads tool descriptions and figures out the sequence itself. MCP is just the standardised pipe that makes that communication possible.

🍽 Analogy — A Restaurant Kitchen

The Host is the head chef (brain of the operation). The MCP Client is the waiter who knows which stations exist. The MCP Servers are the individual stations — grill, salad, dessert. The chef doesn't grill the steak himself; he calls the grill station. The waiter ferries orders back and forth. The chef then plates everything into one final dish for the customer.
